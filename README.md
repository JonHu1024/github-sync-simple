# github-sync-simple
Obsidian同步到Github的简单插件, 不依赖本地的Git, 只需要简单配置Github token, 目前尚未上架Obsidian官方商店, 从Github Gitless Sync改良而来, 在保证同步速度的同时将同步模式的选择权留给用户.

#### 为何开发
- 在同步的时候, 如果我们修改了本地的文件结构(比如建立文件夹归档文件),一般的同步插件往会因为变化过大而放弃同步,为了解决这个问题,同时利用Github存储文本类文件的巨大优势(一般而言比OneDirve等网盘类存储要快很多),我借助Claude和ChatGPT等开发了这个简单的小插件,目前暂无打算上架Obsidian官方商店.

#### 插件特性
- 控制反转(Inverse of Control):这里借用Java Spring框架的术语,意为我们将同步模式的选择权交给用户,由用户自由选择同步的基准(本地、云端、修改时间)，所以我设计了三种模式,不过在确认执行同步之前,我们会先对比当下文件与云端(github)文件的所有不同,并将所要进行的增加、删除、调整操作全部列出,以免意外发生.
- 三个同步模式:
1. push上传:以**本地**的文件结构(包括目录结构)为基准.强制把_云端_的文件结构调整成与**本地**的一模一样.
2. pull拉取:以**云端**的文件结构(包括目录结构)为基准.强制把_本地_的文件结构调整成与**云端**的一模一样.
3. 智能模式:以**最后修改时间新**的文件结构为基准,比对本地和云端的所有文件以及修改时间,删除本地和云端中修改时间旧的文件,保留(从本地上传或者从云端下载)修改时间更新的文件.

#### 使用说明
由于没有添加到Obsidian的官方商店,现在只能手动导入.
1. 下载本项目里的`main.js`和`manifest.json`.
2. 在文件管理器中打开当前Obsidian的工作目录,现实隐藏文件(夹)后进入`.obsidian`目录,在`plugins`目录(没有就自己创建)下创建名为`github-sync-simple`的目录(不要擅自修改).
3. 将下载的`main.js`和`manifest.json`放在`.obsidian/plugins/github-sync-simple`目录下,重启obsidian,在设置里打开本仓库的插件模式后就能看到本插件以及成功加载,此时在Obsidian的最左栏应该可以看到github的图标.
4. 配置:在Github上新建一个私人仓库用于同步,进入本插件的设置界面,然后填写自己的GithubToken(`ghp`开头的那个,这个不清楚可以直接按AI提供的额操作步骤来),以及相关同步仓库即可使用.
